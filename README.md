## Open Source

### phai-run/phai

Founder and maintainer of phai, an MIT-licensed, rules-first personal finance agent: a terminal-first Rust system that puts LLMs on rails. It connects to Pluggy, normalizes banking data into SQLite locally or BigQuery in production, and exposes queryable reports for humans, scripts, dashboards, and agents. The architecture uses a single-binary CLI, a shared FinanceStore trait, embedded migrations, rust_decimal end-to-end, append-only audit events, self-update releases, an MCP surface, and a LiveStore + React local web app embedded into the binary.

The project also carries a full engineering harness: AGENTS.md guardrails, ADRs, privacy rules, TDD, CI, cargo audit, cargo deny, Release Please, and Sentrux quality gates to keep agent-assisted changes reviewable and production-ready.

Technologies: Rust, SQLite, BigQuery, Pluggy, React, LiveStore, MCP, ADRs, agent harness.

Project: https://github.com/phai-run/phai

### feliperun/eai

Rust CLI for natural-language-to-shell command generation: describe the operation, inspect the generated command, confirm, and run. Supports local and low-cost LLM providers, pipe context, command explanation, script and recipe generation, tool discovery, bookmarks, completions, and shell integration.

Technologies: Rust, Ollama, Gemini, Groq, OpenAI, shell UX.

Project: https://github.com/feliperun/eai

### feliperun/dsync

Rust CLI for keeping Markdown as the local source of truth while syncing documents with Google Docs and Linear Docs. It creates or imports documents, maintains frontmatter links, updates Google Docs, Linear, and Git cross-links, and is structured around an extensible provider interface.

Technologies: Rust, Markdown, Google Docs, Linear Docs, provider architecture.
